GitHub和git的相关操作

发表于2021-02-16|更新于2021-05-22|技术GitHub
|字数总计:568|阅读时长:2分钟|阅读量:

一、初始化一个git仓库

  • 将当前文件夹作为git仓库
  • 或者将当前文件夹下的xxx文件夹作为git仓库
  • 初始化后会出现一个名为.git的隐藏文件夹

二、添加远程仓库GitHub

  • 给远程仓库git@github.com:phd99/test.git设置一个别名（远程主机名）test
  • 查看当前设置了哪些远程仓库的别名
  • 删除远程仓库的别名

三、配置验证信息

  • 生成SSH。创建过程中会要求设置密码，可以不设置密码，直接按回车键
  • id_rsa是私钥，id_rsa.pub是公钥，默认位置在C:\Users\用户名\.ssh中。
  • 将公钥添加到github中
    • Settings->SSH and GPG keys->New SSH key
    • 将id_rsa.pub公钥文件中的内容复制到Key中
    • 公钥名称任意
    • 查看是否添加成功

四、克隆远程仓库中的项目

  • 克隆默认分支中的项目
  • 克隆指定分支中的项目

五、下载远程仓库中的项目

  • 下载远程仓库中的项目，并和本地仓库中的项目合并
  • pull和clone的区别
    • clone是从无到有的过程
    • pull是已有本地仓库，想获取最新的项目内容

六、上传文件步骤

6.1. 将内容写入暂存区

  • .表示将文件夹中的所有内容都写入暂存区

6.2. 将暂存区内容添加到本地仓库中

6.3. 上传代码到远程仓库中并合并

  • 格式
    • git push -u <远程主机名> <本地分支名>:<远程分支名>
  • 使用-u后，以后可以直接git push省略了后面的内容
  • 若本地分支名和远程分支名相同，则可以省略远程分支名不写
  • 范例

六、查看当前的本地分支名

七、修改GitHub仓库的默认分支

  • 进入仓库->Settings->Branches->Default branch
